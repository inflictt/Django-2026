Django me template ka use isliye hota hai taaki
business logic (Python code) aur HTML (UI / design) alag-alag rahein.
Isse code clean, readable aur maintainable hota hai.

Overall flow:

User request bhejta hai (URL hit karta hai)

URL request ko view tak le jaata hai

View business logic handle karta hai (data process, DB se data lana, etc.)

View template ko render karta hai

Rendered HTML response user ko wapas bhej diya jaata hai

Template me kya hota hai?

Template me sirf presentation logic hota hai, heavy business logic nahi.

1. Variables

View se data aata hai aur template me show hota hai:

{{ name }}
{{ user.email }}

2. Tags (control logic)

Ye decision making aur looping ke liye hote hain.

if-else

{% if user.is_authenticated %}
  Hello {{ user.username }}
{% else %}
  Please login
{% endif %}


for loop

{% for item in items %}
  {{ item }}
{% endfor %}

3. Filters (data ko modify karna)
{{ name|upper }}
{{ text|length }}
{{ description|truncatewords:10 }}

4. Template inheritance

Common layout reuse karne ke liye:

{% extends "base.html" %}
{% block content %}
  Page content here
{% endblock %}

2) View (views.py)

Yahin business logic hota hai
Data bana ke template ko bhejte hain

from django.shortcuts import render

def home(request):
    name = "Saksham"
    skills = ["Python", "Django", "DSA"]

    return render(request, "home.html", {
        "name": name,
        "skills": skills
    })
